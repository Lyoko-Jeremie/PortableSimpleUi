# Agents

# 实现守则
不使用 `import()` 、 `require()` 动态导入，所有模块静态导入，确保兼容性。
始终保障类型系统正确运行，提高代码的可维护性和可靠性。
特别注意要收窄类型，为IDE提供更准确的类型提示，提升开发效率。

使用jest进行单元测试，确保代码质量和功能正确性。

编写example测试样例，展示如何使用模块功能，并让人工可以快速验证其正确性。

# 验证
使用 `yarn run build:examples` 编译示例来检查examples目录下的代码是否正确，确保示例代码能够正常运行并展示预期的功能。
使用 `yarn run test` 运行单元测试，确保所有测试用例都通过，验证代码的正确性和稳定性。
使用 `yarn run build` 构建项目，确保代码能够成功编译并生成可用的输出文件，验证构建过程的正确性和完整性。
